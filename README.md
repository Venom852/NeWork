# Дипломный проект по профессии «Android Developer»

### Цель проекта

Цель дипломного проекта — создать приложение для Android. Приложение представляет из себя социальную сеть, которая позволит пользователям создавать посты и события, указывать свои места работы.

### Инструменты используемые в процессе разработки дипломного проекта

- Android Studio
- Kotlin
- RecyclerView
- Okhttp
- Retrofit.
- PagingData
- Ksp
- Gson
- SwipeRefreshLayout
- Dagger Hilt
- Swagger
- Figma
- Git + GitHub
- SQL + SQLite
- Room
- Glide
- MapKit

### Краткое описание функционала

*1. Основной экран приложения.*

Стартовый экран содержит:
- нижнее меню с тремя кнопками: посты, события и пользователи,
- кнопку меню в AppBar с возможностью перехода на вход в аккунт и регистрацию или просмотр профиля.

![](/pic/main_screen.png)

*1.1. Экран со списком постов.*

Этот фрагмент главного экрана содержит:
- список постов,
- кнопку +, нажатие которой ведёт на создание нового поста или диалог с предложением войти/зарегистрироваться.

Карточка поста включает:
- [x] аватар автора поста;
- [x] имя автора;
- [x] дату публикации в формате `dd.mm.yyyy HH:mm`;
- [x] кнопку лайка с количеством лайков;
- [x] вложение, при наличии: аудио, видео или фото;
- [x] ссылку, при наличии, с переходом в браузер; 
- [x] кнопку вызова меню с возможностью удаления или перехода к редактированию, в случае, если текущий пользователь является автором;
- [x] текст поста;
- [x] текст с названием аудио;
- [x] инормацию о длительности аудио;
- [x] кнопку поделиться постом;
- [x] кнопку просмотра видео;
- [x] кнопку проигрывания музыки.

![](/pic/screen_with_list_of_post.png)

При нажатии на карточку поста происходит переход к детальному виду с отображением дополнительной информации:
- [x] список упомянутых юзеров;
- [x] последнее место работы автора, либо текст «В поиске работы»;
- [x] кнопку перехода назад;
- [x] карту с маркером, при наличии координат.

*2. Экран входа в приложение.*

Содержит поля ввода для логина и пароля.

Для полей установлены требования:
- [x] логин — непустая строка;
- [x] пароль — непустая строка.

При несоответствии требованиям в полях отображаются информативные сообщения с возможностью исправления данных.

При ошибочном переходе на фрагмент содержит кнопку перехода назад, а так же кнопку с возможностью перейти на экран регистрации.

В случае получения от сервера 400 кода отображается ошибка в виде Toast «Неправильный логин или пароль».

В случае получения от сервера 404 кода отображается ошибка в виде Toast «Пользователь незарегистрирован».

![](/pic/fragment_screen_with_sign_in.png)

*3. Экран регистрации.*

Содержит поля ввода для логина, имени, пароля и повтора пароля. А также кнопку с возможностью выбора фото и её превью.

Для полей установлены требования:
- [x] логин — непустая строка;
- [x] имя — непустая строка;
- [x] пароль — непустая строка;
- [x] аватар — изображение в формате jpeg, png, максимальный размер 2048*2048.

При несоответствии требованиям в полях отображаются информативные сообщения с возможностью исправления данных.

При ошибочном переходе на фрагмент содержит кнопку перехода назад, а так же кнопку с возможностью перейти на экран регистрации.

В случае получения от сервера 403 кода отображается ошибка в виде Toast «Пользователь с таким именем уже зарегистрирован».

В случае получения от сервера 404 кода отображается ошибка в виде Toast «Некорректный формат фото».

![](/pic/fragment_screen_with_sign_up.png)

*4. Экран создания/редактирования поста.*

На этот экран может попасть только авторизованный пользователь.

Экран содержит:
- поле ввода текста;
- кнопку выбора локации: переход на фрагмент с картой;
- кнопку выбора упомянутых пользователей;
- кнопки выбора изображения: фото или галерея;
- кнопки выбора вложения: аудио, видео;
- кнопку выбора локации;
- кнопку перехода назад;
- кнопку сохранить в AppBar.

После создания поста следует возврат назад к списку постов.
